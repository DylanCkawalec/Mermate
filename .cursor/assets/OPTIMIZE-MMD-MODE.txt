## Mission
Take the existing markdown or Mermaid and improve it into a stronger architecture representation without breaking the original design intent.

This is not a rewrite-for-style task.
This is a structure-preserving optimization task.

## Required operating behavior
1. First understand what the current source is trying to represent.
2. Determine whether it is:
   - mostly correct but flat
   - mostly correct but cluttered
   - structurally weak
   - invalid Mermaid
   - under-labeled
   - overloaded and needing multiple diagrams
3. Preserve:
   - entities
   - directionality
   - architecture intent
   - key failure paths
   - important labels and semantics
4. Improve:
   - decomposition
   - readability
   - boundary clarity
   - Mermaid correctness
   - node/edge discipline
   - operational realism
5. If the current diagram should be split into multiple views, do so deliberately rather than forcing everything into one overloaded graph.
6. Render repeatedly in cheap/default mode to compare versions.
7. Assess returned diagrams critically.
8. Only use Max mode for the final optimized version once the architecture is already strong.

## Optimization rules
- Do not introduce breaking conceptual changes.
- Do not change the system into a different architecture.
- Do not add entities unless strongly implied or necessary for correctness.
- Do not flatten layered systems.
- Do not turn a valid focused diagram into a noisy “bigger” one.
- Prefer structural truth over decorative complexity.

## Required Mermate loop
- analyze current source
- identify structural weaknesses
- strengthen the draft
- render cheaply
- compare against the original
- iterate until materially better
- finalize with Max only if useful

## Required outputs
Return:
1. Executive summary
2. Current-source assessment
3. What must be preserved
4. What should be improved
5. Optimization plan
6. Iterative refinement log
7. Cheap/default render comparisons
8. Final optimized source
9. Final render mode used
10. Final artifact summary
11. Final verdict on whether the optimized diagram is truly better without breaking the original design

## Judgment standard
A successful optimization:
- preserves the original design intent
- improves readability and structure
- improves correctness or Mermaid discipline
- improves engineering usefulness
- does not introduce unnecessary changes
